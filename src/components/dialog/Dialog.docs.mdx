# Dialog

This component is used to show an overlay dialog on the page. It supports different sizes and position behaviours on mobile and desktop.

This component uses the `Dialog` component from ReachUI. To install:

```
npm i @reach/dialog
```

The following CSS snippet needs to be added to the root css file of the project:

```css
:root {
  --reach-dialog: 1;
}
```

### Subcomponents

You can use the following components for the different areas of the dialog (see examples below):

- Base components
  - `DialogHeaderArea`
  - `DialogHeaderLeftArea`
  - `DialogHeaderRightArea`
  - `DialogHeaderTitle`
  - `DialogContentArea`
  - `DialogFooterArea`
  - `DialogFooterActions`
- Convienience components
  - `DialogHeader`
  - `DialogHeaderClose`

**Note:** The component does not include the button. The button it is just used to activate the dialog changing the value of the prop `isOpen`.

### Theme

- `dialog.overlay.base`: Define the base styles of the overlay
- `dialog.overlay.mobilePosition[$position]`: Define the styles of the dialog for the given mobile position.
- `dialog.overlay.desktopPosition[$position]`: Define the styles of the dialog for the given desktop position.
- `dialog.dialog.base`: Define the base styles of the dialog window.
- `dialog.dialog.mobilePositionSize[$position].base`: Define the base styles of the dialog for the given mobile position.
- `dialog.dialog.mobilePositionSize[$position][$size]`: Define the styles of the dialog for the given mobile position and size.
- `dialog.dialog.desktopPositionSize[$position].base`: Define the base styles of the dialog for the given desktop position.
- `dialog.dialog.desktopPositionSize[$position][$size]`: Define the styles of the dialog for the given desktop position and size.
